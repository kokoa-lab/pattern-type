# PatternType — 디자인 패턴 성격 유형 문서

## 개요

PatternType은 12개의 코딩 상황 질문을 통해 사용자의 코딩 성격을 7가지 디자인 패턴 중 하나로 진단하는 심리테스트입니다.

---

## 1. Observer 👁️ — 변화의 감지자

**설명:** 주변 코드의 변화에 민감하게 반응하며, 이벤트 드리븐 사고를 합니다. 실시간 데이터 흐름과 상태 변화를 직관적으로 파악하는 능력이 뛰어납니다.

**업무 스타일:** 변화를 먼저 감지하고, 팀에 빠르게 알리는 얼리어답터

### 강점
- 실시간 시스템 설계
- 반응형 프로그래밍
- 상태 관리

### 주의점
- 과도한 구독으로 인한 복잡성
- 디버깅 어려움

### 궁합 패턴
- **Strategy** — 유연한 전략 교체와 이벤트 반응이 시너지를 냄
- **Adapter** — 외부 이벤트를 내부 형식으로 변환하는 데 조화로움

---

## 2. Singleton 👑 — 유일무이한 존재

**설명:** 독립적이고 일관된 방식을 고수합니다. 전역적인 관점에서 시스템을 바라보며, 중복을 극도로 싫어합니다.

**업무 스타일:** 한 번 정한 원칙은 끝까지 지키는 신뢰의 아이콘

### 강점
- 일관성 유지
- 리소스 최적화
- 전역 상태 관리

### 주의점
- 테스트 어려움
- 결합도 증가 가능성

### 궁합 패턴
- **Builder** — 체계적인 구축과 일관된 인스턴스 관리의 조합
- **Factory** — 생성 로직을 중앙에서 관리하는 공통 철학

---

## 3. Factory 🏭 — 창조의 대가

**설명:** 다양한 객체와 솔루션을 효율적으로 생산합니다. 문제 해결을 위한 여러 변형을 빠르게 만들어내는 능력이 탁월합니다.

**업무 스타일:** 아이디어를 빠르게 프로토타입하는 메이커

### 강점
- 객체 생성 캡슐화
- 다형성 활용
- 유연한 인스턴스화

### 주의점
- 과도한 추상화 경향
- 클래스 폭발

### 궁합 패턴
- **Strategy** — 생성된 객체에 다양한 전략을 적용하는 강력한 조합
- **Builder** — 복잡한 객체를 단계별로 팩토리에서 생산

---

## 4. Strategy 🎯 — 전략의 마스터

**설명:** 상황에 따라 최적의 알고리즘을 선택합니다. 유연한 사고와 적응력으로 다양한 문제에 대응합니다.

**업무 스타일:** 상황 판단이 빠르고, 플랜 B를 항상 준비하는 전략가

### 강점
- 알고리즘 교체 용이
- 조건문 제거
- 런타임 유연성

### 주의점
- 전략 선택 로직 복잡화
- 클래스 수 증가

### 궁합 패턴
- **Factory** — 전략 객체를 동적으로 생성하는 완벽한 파트너
- **Observer** — 상태 변화에 따라 전략을 자동 전환

---

## 5. Decorator 🎨 — 기능의 연금술사

**설명:** 기존 것을 해치지 않으면서 새로운 가치를 더합니다. 점진적 개선과 확장을 통해 시스템을 발전시킵니다.

**업무 스타일:** 기존의 좋은 것을 더 좋게 만드는 개선의 달인

### 강점
- 기존 코드 수정 없이 확장
- OCP 원칙 준수
- 조합 가능한 기능

### 주의점
- 래퍼 지옥
- 디버깅 시 레이어 추적 어려움

### 궁합 패턴
- **Adapter** — 래핑과 변환의 달인들이 만나면 호환성 문제가 사라짐
- **Builder** — 데코레이터를 단계별로 쌓아가는 체계적 접근

---

## 6. Adapter 🔌 — 연결의 중재자

**설명:** 서로 다른 인터페이스 사이에서 소통을 가능하게 합니다. 호환성 문제를 우아하게 해결하는 능력이 있습니다.

**업무 스타일:** 다른 팀/시스템과의 협업을 매끄럽게 만드는 조율자

### 강점
- 인터페이스 변환
- 레거시 통합
- 시스템 간 브릿지

### 주의점
- 추가 레이어 오버헤드
- 과도한 어댑터 계층

### 궁합 패턴
- **Decorator** — 기능 래핑과 인터페이스 변환의 환상적 콜라보
- **Observer** — 이벤트 형식을 자유롭게 변환하며 연결

---

## 7. Builder 🏗️ — 체계적인 건축가

**설명:** 복잡한 것도 단계별로 차근차근 완성합니다. 명확한 프로세스와 가독성 높은 코드를 추구합니다.

**업무 스타일:** 계획을 세우고 단계별로 실행하는 체계적인 실행가

### 강점
- 복잡한 객체 생성
- 가독성 높은 코드
- 불변 객체 생성

### 주의점
- 보일러플레이트 코드
- 단순 객체에는 과도함

### 궁합 패턴
- **Factory** — 빌더가 만든 설계도를 팩토리가 대량 생산
- **Singleton** — 유일한 인스턴스를 체계적으로 구축

---

## 점수 산정 방식

각 질문의 선택지에는 7가지 패턴별 점수(0~3)가 부여됩니다. 12개 질문 완료 후 가장 높은 누적 점수를 가진 패턴이 사용자의 결과 타입이 됩니다.

## 결과 공유

결과는 URL 파라미터(`?result=패턴명`)로 공유 가능하며, 이미지 저장 기능도 지원합니다.
